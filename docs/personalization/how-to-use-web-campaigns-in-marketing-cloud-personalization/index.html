How to create a Web Campaign in Marketing Cloud Personalisation

Once you installed the SDK into your website, built the sitemap with a content zone, you are now able to create a Web Campaign in Marketing Cloud Personalisation.

Go on your website, and turn on the visual editor of the Marketing Cloud Personalisation Chrome Extension to see the controller (Octagon shape bottom right of your screen).

Click on “Web Campaign” to get started and select Global Templates at the top left of the pop-up screen.

What are Global Templates​

Global Templates are the easiest way to build a website template as they provide a base that you can modify. Marketing Cloud Personalisation provides 14 Global Templates. All of them have different features and purposes.

In this article we are going to use “Einstein Content Recommendations” as we want to promote personalised articles to our website visitors.

Once you selected the Global Template, click “Clone email Template” to populate the code snippet related and start configuring it for your content zone.

As mentioned above, setting-up the content zone is required prior to building the web template.

Creating an Einstein Recipe​

The very first step before configuring your Web Template is to create an Einstein Recipe. An Einstein Recipe is composed of one or several parameters that will determine the content that should be displayed on your web template.

In Personalisation, go to Einstein Recipes and click New Recipe. After entering a name, you can select different ingredients (parameters) that will impact what are the articles displayed in the web template.

Select the object type that this recipe is for. If my recipe is for the catalog object Article, then I need to select Article in the drop down menu Recommendation Type at the top right of my screen.

As you just implemented the sitemap, and don't have any data history, my recommendation at first is to start with simple Ingredients, such as Any Eligible Item.

Then you can click Simulate to make sure your recipe is working correctly. Make sure that the recipe is displaying information (image, title, etc), after that you can hit the button Publish!

Configuring your Web Template​

The first step after cloning your global template is to select the Content Zones in Template (located at the bottom left of your screen).

Then you must select the recipe that you built in the step above.

danger

If you don’t see your recipe it is because you didn’t created a recipe for the correct object type

Also you can see that there is a console at the bottom of your screen that contains the tabs Handlebars, CSS, Clientside Code, Serverside Code. All these sections are needed to build your web template.

We click Save & publish at the top of the screen to see how the global template looks like on our website and what are the modifications required to match our expectations.

Then you should be able to select a specific type of object (From Catalog) in the ServerSide code tab of the Console. In our case we are going to use the Catalog object Article. (restrictItemType and return)

import { RecommendationsConfig, recommend } from "recs";

export class EinsteinContentRecsTemplate implements CampaignTemplateComponent {

    /**
     * Developer Controls
     */

    @hidden(true)
    maximumNumberOfItems: 2 | 4 | 6 = 4;

    /**
     * Business-User Controls
     */

    @title("Recommendations Block Title")
    header: string = "Title Text";

    @title(" ")
    recsConfig: RecommendationsConfig = new RecommendationsConfig()
        .restrictItemType("Article")
        .restrictMaxResults(this.maximumNumberOfItems);

    @header('Recommendation Display Options')

    @title("Show content name")
    nameVisibility: boolean = true;

    @title("Show content description")
    descriptionVisibility: boolean = true;

    run(context:CampaignComponentContext) {
        this.recsConfig.maxResults = this.maximumNumberOfItems;

        return {
            itemType: this.recsConfig.itemType,
            articles: recommend(context, this.recsConfig)
        }
    }

}

Make sure to update the object also in Handlebars (each variable):

<section class="evg-einstein-content-recs" data-evg-campaign-id="{{campaign}}" data-evg-experience-id="{{experience}}"
data-evg-user-group="{{userGroup}}" aria-label="Content Recommendations{{#if header}} - {{header}}{{/if}}">
    {{#if header}}
    <h3 class="evg-header evg-h3">
        {{~header~}}
    </h3>
    {{/if}}
    <div class="evg-content-recs">
        {{~#each articles}}
        <div class="evg-container">
            <article class="evg-content-rec" data-evg-item-id="{{id}}" data-evg-item-type="{{../itemType}}">
                <a href="{{attributes.url.value}}">
                    <img class="evg-content-img" src="{{attributes.imageUrl.value}}" alt="{{attributes.name.value}}"/>
                </a>
                <div class="evg-content-details">
                    {{#if ../nameVisibility}}
                    <a class="evg-content-title" href="{{attributes.url.value}}">
                        {{~attributes.name.value~}}
                    </a>
                    {{/if}}
                    {{#if ../descriptionVisibility}}
                        {{#if attributes.description.value}}
                        <div class="evg-content-description">
                            {{~attributes.description.value~}}
                        </div>
                        {{/if}}
                    {{/if}}
                </div>
            </article>
        </div>
        {{/each~}}
    </div>
</section>

Finally, make sure that you are using the correct Attributes in your Handlebars code. As you may have to adapt the variable that you are calling.

Once you are able to pull the variables from your object catalog (Article), you can design your content zone in the CSS tab of the console.

danger

Sometimes the web template doesn’t update/render instantly so you have to browse your website and come back to the content zone to force the webpage to load with your content.

Create the web campaign​

Great! You now created your Einstein recipe & web template. Now we are ready to build the Web Campaign. The idea behind the Web Campaign is to define when we should trigger the Web Template in the content zone.

Navigate to your website and click on the Marketing Cloud Personalisation, then turn on the Visual Editor button. Once you turn on this button, you should see the octagon at the bottom right of your screen, click New Campaign.

After giving a name to your campaign you can select a rule (parameters to trigger the web template into the content zone). In our case we are selecting something pretty simple: Browser (As Source) = Google Chrome (As Value) (As I’m using Google Chrome I should be able to visualize the campaign).

Then I will click on campaign type A/B Test, and push the experience 1 at 100% so that I’m sure that my Web Template will be triggered when I meet the criteria (rules) related to the campaign.

Finally, I just need to click Add a Template, and select the Web Template that we created in the step above.

Click Save & Publish the campaign. You should now be able to see the campaign on your website.
