How to use Open Time Email in Marketing Cloud Personalisation

What is Open Time Email​

Open Time Email is a feature of Marketing Cloud Personalisation that allows users to send dynamic and personalised email communications. Long story short, recipients will receive an email with different information based on when they open it.

A basic example could be : If user A opens email at 9:00 am, the email title will display “Good Morning User A” If user A opens email at 6:00 pm, the email title will display “Good Evening User A”.

Now let’s imagine that you are an e-commerce store owner, you could go further: If user A opens email at 9:00 am, the email title will display “Good Morning User A” and will display a coffee cup in product recommendation. If user A opens email at 6:00 pm, the email title will display “Good Evening User A and will display a pillow in product recommendation”

What impresses me the most about the Open Time Email feature of Marketing Cloud Personalisation is the possibility to personalize content AFTER sending the email. When the recipient opens it.

Imagine back in time when people were sending letters to each other if they were capable of sending a letter that contains different text based on when you open it. That would be pure magic.

What are the components of Open Time Email​

3 components are needed to create an Open Time Email: Open Time Email Template Open Time Email Campaign Einstein Recipe

Basically, the Open Time Email Template is the space in Marketing Cloud Personalisation where you create and design a content block for a specific asset (a product, a blog post etc.). The Einstein Recipe is where you establish rules such as displaying THIS type of products/blog posts to THAT type of users - And Open Time Email Campaign is where you merge the Open Time Email Template and the Recipe to create a block of HTML that you can integrate to your email template in Marketing Cloud.

Let’s deep dive into each of these components

Open Time Email Templates​

Open Time email template is pretty straightforward. You can simply create a content block with HTML and CSS, and use personalization (javascript) to have dynamic variables. For example you DO NOT want to create a content block for each product. You create 1 content block for Products that will dynamically adapt to each product of your store. Here’s the base code of a product content block:

<div style="padding:10px; text-align:center; font-family: Arial;">
    <img src="${item.imageUrl}" style="max-width:180px; max-height:150px;">
    <div style="text-align:left;">
        ${item.name}
    </div>
    <div style="margin-top:5px; text-align:left;">
        $currency.format(${item.price})
    </div>
    <div style="position:absolute; left: 0px; right: 0px; bottom: 10px;">
        <button style="border:none; padding: 10px; background:#1D73C9; color:#ffffff; font-size:16px;">
            Buy now »
        </button>
    </div>
</div>

As you can see this snippet of code is: Displaying the name of the product: ${item.name} Displaying the price of the product: $currency.format(${item.price}) Displaying a button: Buy now

How to add attributes and dynamic variables​

Something quite interesting to know is that Marketing Cloud Personalisation will render the content block you coded into an image. So in the code snippet the function of the button is purely visual as wherever the user clicks on the image he will be redirected to the same link.

My question at first was how to find variables such as ${item.name} as the product (or other asset) that you are trying to display can contain many attributes. You just have to go in Catalog > Products Click on one of the products you added (or blog post or whatever object) and then Identify the field you are trying to add.

Here’s a few variables that you can include in your Email Open Time Content Block: 

// ID
${item.id}                              // Returns Product ID
${item.attributes.published.value}      // Error generating preview image: Unfulfilled variable
${item.custom.customDatetime}           // Error generating preview image: Unfulfilled variable

// Name
${item.name}                            // Returns Product Name
${item.attributes.name.value}           // Returns Product Name
${item.custom.name}                     // Returns Product Name

// Published Date
${item.published}                       // Returns Product Published Date
${item.attributes.published.value}      // Returns Product Published Date
${item.custom.published}                // Error generating preview image: Unfulfilled variable

How to add IF/ELSE conditions​

Second thing that I wish I knew when starting Open Time Email is IF/ELSE conditions. Using that is extremely useful if you need to display items in your template only in specific cases. For example, I create a content block for a product and I want to display an icon “NEW” on it ONLY if I added the attribute NEW == True in the object catalog then the IF/ELSE condition is required.

Here’s an example of IF/ELSE condition. Of course, you also use only IF. 

<img src="${item.imageUrl}" style="max-width:40px; max-height:40px;">

GoodBye My Friend!

How to add Fallback value​

Fallback values are also useful for the case of there is no value when you are trying to display a specific dynamic variable. Let’s imagine you created a content block for your products, and everything works perfectly fine, except for one product where the image is missing. Do you really want to display a blank image? NO! That’s why using fallback value is important. It’s like in case there is no data (null), you can display a default value.

Here’s the an example of fallback value: 

#field(${item.imageUrl}", 'FallBack Value')

In the code snippet above if there is no image URL we are displaying “FallBack Value”
